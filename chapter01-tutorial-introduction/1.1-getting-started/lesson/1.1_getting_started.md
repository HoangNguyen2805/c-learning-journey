# 1.1 - Getting Started

## Big idea

A C program does not run directly from the `.c` file.

The basic workflow is:

```text
write code -> save -> compile -> run
```

---

## Smallest useful C program

```c
#include <stdio.h>

int main() {
    printf("Hello, C!\n");

    return 0;
}
```

---

## `#include <stdio.h>`

```c
#include <stdio.h>
```

This gives the program access to standard input/output tools.

For now, the important tool is:

```c
printf()
```

`printf()` prints text to the screen.

---

## `main`

```c
int main()
```

`main` is where a C program starts running.

When the program runs, C looks for the `main` function first.

---

## Curly braces `{ }`

```c
int main() {
    // code goes here
}
```

Curly braces show the body of the function.

```text
{ = start
} = end
```

Everything inside `{ }` belongs to `main`.

---

## `printf`

```c
printf("Hello, C!\n");
```

`printf` prints text to the screen.

The text inside double quotes is called a string.

---

## `\n`

```c
\n
```

`\n` means newline.

It moves the output to the next line.

Example:

```c
printf("Hello\n");
printf("C\n");
```

Output:

```text
Hello
C
```

---

## Semicolon `;`

Most normal C statements end with a semicolon.

Example:

```c
printf("Hello, C!\n");
return 0;
```

The semicolon means the statement is finished.

---

## `return 0`

```c
return 0;
```

This ends the `main` function.

Returning `0` usually means the program finished successfully.

---

## Source code vs executable

The `.c` file is the source code file.

Example:

```text
1.1_getting_started.c
```

This is the file humans write and read.

After compiling, gcc creates an executable program.

Example:

```text
1.1_getting_started
```

This is the program the computer can run.

---

## Compile

Command:

```bash
gcc 1.1_getting_started.c -o 1.1_getting_started
```

Meaning:

```text
gcc = C compiler
1.1_getting_started.c = input source code file
-o = output name
1.1_getting_started = executable program name
```

The compiler turns C source code into a program the computer can run.

Flow:

```text
1.1_getting_started.c -> gcc -> 1.1_getting_started
```

---

## Run

Command:

```bash
./1.1_getting_started
```

Meaning:

```text
./ = look in the current folder
1.1_getting_started = program to run
```

So:

```bash
./1.1_getting_started
```

means run the executable program named `1.1_getting_started` from the current folder.

---

## What I need to remember

```text
#include <stdio.h> = lets me use printf

main() = program starts here

printf() = prints text

\n = newline

; = ends a statement

return 0 = program finished successfully

gcc file.c -o program = compile

./program = run
```
