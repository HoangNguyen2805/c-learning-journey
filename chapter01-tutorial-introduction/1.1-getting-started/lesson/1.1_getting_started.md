# 1.1 Getting Started

## The Big Picture

A C program has a simple lifecycle:
The **source code** (what you write) is different from the **executable** (what the computer runs).

---

## The Smallest Useful C Program

Here's the bare minimum:

```c
#include <stdio.h>

int main() {
    printf("Hello, C!\n");
    return 0;
}
```

This program:
1. Includes the standard input/output library
2. Defines a `main` function (where execution starts)
3. Prints "Hello, C!" to the screen
4. Returns 0 (success)

---

## Breaking Down Each Part

### `#include <stdio.h>`

```c
#include <stdio.h>
```

This tells C: "I want to use tools from the standard I/O library."

**What is a library?**
A library is a collection of pre-written functions you can use.

**What is `stdio.h`?**
- `stdio` = "standard input/output"
- `.h` = header file
- It contains functions like `printf()` and `scanf()`

**Without this line, `printf()` won't work.**

---

### `int main()`

```c
int main()
```

This declares the `main` function. Key facts:

- **`main` is special** — every C program must have a `main` function
- **Program execution starts here** — C looks for `main()` and runs it first
- **`int` means return type** — `main` returns an integer (0 for success, non-zero for error)

---

### Curly Braces `{ }`

```c
int main() {
    // code goes here
}
```

Curly braces mark the **body** of a function:
- `{` = start of function body
- `}` = end of function body
- Everything between them belongs to `main`

---

### `printf()`

```c
printf("Hello, C!\n");
```

`printf()` prints text to the screen.

**Anatomy:**
- `printf` = function name (short for "print formatted")
- `(...)` = arguments inside parentheses
- `"Hello, C!\n"` = a string (text in double quotes)
- `;` = ends the statement

---

### The Newline `\n`

```c
\n
```

`\n` is an **escape sequence** meaning "newline."

**Without `\n`:**
```c
printf("Hello");
printf("World");
```
Output: `HelloWorld`

**With `\n`:**
```c
printf("Hello\n");
printf("World\n");
```

---

### The Semicolon `;`

```c
printf("Hello!\n");
return 0;
```

Semicolons end statements. Think of them as periods in English.

**Without semicolons, the compiler gets confused.**

---

### `return 0;`

```c
return 0;
```

This ends the `main` function and sends a value back to the operating system.

- **`0` = success** — the program ran without errors
- **Non-zero = error** — something went wrong

For example:
```c
return 1;  // Signal an error
return 2;  // Signal a different error
```

## Source Code vs Executable

| Type | File Extension | What It Is | Who Reads It |
|------|----------------|-----------|---------------|
| Source Code | `.c` | Human-readable C code | You and other programmers |
| Executable | (none or `.exe`) | Compiled program | The computer |

**Example:**
- `hello.c` = source code (you can open in editor)
- `hello` = executable (you can run with `./hello`)

## Running the Program

**Command:**
```bash
./hello
```

**Breaking it down:**
- `./` = "in the current directory"
- `hello` = the program to run

**Alternative:**

is equivalent to:

## Common Mistakes

### ❌ Mistake 1: Forgetting `#include <stdio.h>`
```c
int main() {
    printf("Hello\n");  // ERROR: printf not defined
    return 0;
}
```
**Fix:** Add `#include <stdio.h>` at the top

### ❌ Mistake 2: Forgetting semicolons
```c
printf("Hello\n")
return 0;
```
**Error:** `error: expected ';' before 'return'`

**Fix:** Add `;` after `printf()`

### ❌ Mistake 3: Wrong quotes
```c
printf('Hello\n');  // Single quotes are wrong
```
**Fix:** Use double quotes: `printf("Hello\n");`

### ❌ Mistake 4: Trying to run the `.c` file
```bash
./program.c  # WRONG
```
**Fix:** Compile first, then run the executable:
```bash
gcc program.c -o program
./program
```

## Key Takeaways

| Concept | Meaning |
|---------|---------|
| `#include <stdio.h>` | Give my program access to `printf()` |
| `int main()` | Start of the program |
| `{ }` | Function body |
| `printf()` | Print text to screen |
| `\n` | Newline (move to next line) |
| `;` | End of statement |
| `return 0;` | Program finished successfully |
| `gcc file.c -o prog` | Compile source code |
| `./prog` | Run the executable |
