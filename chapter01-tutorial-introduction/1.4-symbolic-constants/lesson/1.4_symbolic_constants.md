# 1.4 - Symbolic Constants

## Big idea

This section introduces symbolic constants using `#define`.

A symbolic constant gives a name to an important value.

Instead of writing numbers directly in the code, we give those numbers names.

---

## Example

```c
#define LOWER 0
#define UPPER 300
#define STEP 20
```

Meaning:

```text
LOWER = starting value
UPPER = ending value
STEP = amount added each time
```

---

## Why use symbolic constants?

Without symbolic constants:

```c
for (fahr = 0; fahr <= 300; fahr = fahr + 20)
```

This works, but the numbers are not explained.

With symbolic constants:

```c
for (fahr = LOWER; fahr <= UPPER; fahr = fahr + STEP)
```

This is easier to understand because the names explain the purpose of the numbers.

---

## Full program

```c
#include <stdio.h>

#define LOWER 0
#define UPPER 300
#define STEP 20

int main() {
    int fahr;

    printf("Fahrenheit Celsius\n");

    for (fahr = LOWER; fahr <= UPPER; fahr = fahr + STEP) {
        printf("%3d %6d\n", fahr, 5 * (fahr - 32) / 9);
    }

    return 0;
}
```

---

## Important rules for `#define`

`#define` does not use `=`.

Correct:

```c
#define LOWER 0
```

Wrong:

```c
#define LOWER = 0
```

`#define` does not use a semicolon.

Correct:

```c
#define UPPER 300
```

Wrong:

```c
#define UPPER 300;
```

---

## Symbolic constants vs magic numbers

Magic numbers are numbers written directly in code without a clear meaning.

Less clear:

```c
fahr <= 300
```

More clear:

```c
fahr <= UPPER
```

Now we know `UPPER` means the ending temperature.

## Small note about `const`

C also has `const` variables.

Example:

```c
const int MAX_SCORE = 100;
```

But K&R Chapter 1.4 mainly teaches `#define`.

For now:

```text
#define = symbolic replacement before compile
const = typed variable that cannot be changed
```

We saved the deeper `const` lesson for Chapter 2.
