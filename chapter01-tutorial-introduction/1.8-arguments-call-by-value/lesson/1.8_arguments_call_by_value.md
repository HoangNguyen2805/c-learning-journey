# 1.8 - Arguments: Call by Value

## Big idea

This section introduces **call by value**.

In C, when we pass a value into a function, the function receives a **copy** of that value.

The function does not receive the original variable.

---

## Argument vs parameter

Example:

```c
changeNumber(number);
```

`number` is the argument.

It is the value sent into the function.

---

Function header:

```c
void changeNumber(int x)
```

`x` is the parameter.

It receives a copy of the argument.

---

## Call by value

Call by value means:

```text
The function gets a copy of the value.
```

So if the function changes the parameter, it only changes the copy.

The original variable in `main` does not change.

---

## Example program

```c
#include <stdio.h>

void changeNumber(int x);

int main() {
    int number;

    number = 5;

    printf("Before function call: %d\n", number);

    changeNumber(number);

    printf("After function call: %d\n", number);

    return 0;
}

void changeNumber(int x) {
    x = 100;

    printf("Inside function: %d\n", x);
}
```

---

## Output

```text
Before function call: 5
Inside function: 100
After function call: 5
```

---

## Why does `number` stay 5?

When this line runs:

```c
changeNumber(number);
```

C copies the value of `number` into `x`.

So inside the function:

```text
x = 5
```

Then the function changes `x`:

```c
x = 100;
```

But `x` is only a copy.

The original variable `number` in `main` is still 5.

---

## K&R idea

K&R explains that function arguments in C are passed by value.

This means a function can safely change its parameter without changing the original variable in the caller.

---

## Important note

Later, arrays and pointers can behave differently.

But for normal variables like `int`, `double`, and `char`:

```text
C passes a copy.
```

---

## What I need to remember

```text
argument = value sent into a function

parameter = variable inside the function

call by value = function receives a copy

changing the parameter does not change the original variable

normal variables are passed by value in C
```
