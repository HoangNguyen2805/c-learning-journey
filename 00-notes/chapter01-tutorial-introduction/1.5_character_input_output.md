# 1.5 - Character Input and Output

K&R 1.5 introduces character input and output.

This section uses:

```text
getchar()
putchar()
EOF
while loop
character counting
line counting
word counting preview
```

Important: Chapter 1 is a quick tour. It uses `while` before fully teaching loops. Loops are explained deeper later in Chapter 3.

---

## Big idea

Instead of reading a full number or word like `scanf`, `getchar()` reads input **one character at a time**.

```c
c = getchar();
```

`putchar()` prints one character.

```c
putchar(c);
```

So:

```text
getchar() = read one character
putchar(c) = print one character
```

---

## Why use int instead of char?

K&R usually writes:

```c
int c;
```

not:

```c
char c;
```

because `getchar()` can return either:

```text
a real character
or EOF
```

`EOF` means “end of file” or “no more input.”

`EOF` is not a normal character, so `c` should be an `int`.

---

## Simple getchar example

```c
#include <stdio.h>

int main() {
    int c;

    c = getchar();

    putchar(c);

    return 0;
}
```

If the user types:

```text
A
```

The program prints:

```text
A
```

---

## Copy input to output

K&R shows this pattern:

```c
#include <stdio.h>

int main() {
    int c;

    while ((c = getchar()) != EOF) {
        putchar(c);
    }

    return 0;
}
```

This means:

```text
read one character
if it is not EOF, print it
repeat
```

---

## Important line

```c
while ((c = getchar()) != EOF)
```

This line does two jobs:

```text
1. getchar() reads one character
2. the character is stored in c
3. c is compared with EOF
```

The extra parentheses matter:

```c
(c = getchar())
```

means:

```text
store the character into c first
```

Then:

```c
!= EOF
```

means:

```text
keep going while c is not EOF
```

---

## Newline is also a character

When you press Enter, C sees a newline character:

```c
'\n'
```

So if you type:

```text
A
```

and press Enter, the input actually includes:

```text
A
\n
```

That is why character programs may seem to read the Enter key too.

---

## Counting characters

Example:

```c
#include <stdio.h>

int main() {
    int c;
    int count = 0;

    while ((c = getchar()) != EOF) {
        count++;
    }

    printf("Characters: %d\n", count);

    return 0;
}
```

This counts every character typed, including spaces and newlines.

---

## Counting lines

A line ends when the program sees:

```c
'\n'
```

Example:

```c
#include <stdio.h>

int main() {
    int c;
    int lines = 0;

    while ((c = getchar()) != EOF) {
        if (c == '\n') {
            lines++;
        }
    }

    printf("Lines: %d\n", lines);

    return 0;
}
```

---

## Counting blanks, tabs, and newlines

Special characters:

```text
' '  = blank space
'\t' = tab
'\n' = newline
```

Example:

```c
if (c == ' ') {
    blanks++;
}

if (c == '\t') {
    tabs++;
}

if (c == '\n') {
    newlines++;
}
```

---

## EOF

`EOF` means end of input.

On Linux/Mac/WSL terminal, send EOF with:

```text
Ctrl + D
```

On Windows Command Prompt, EOF is usually:

```text
Ctrl + Z then Enter
```

In WSL, use:

```text
Ctrl + D
```

---

## Difference between scanf and getchar

`scanf` reads formatted input.

Example:

```c
scanf("%d", &age);
```

This is good for reading:

```text
int
double
string
```

`getchar()` reads one character at a time.

Example:

```c
c = getchar();
```

This is good for:

```text
copying input
counting characters
counting lines
processing text one character at a time
```

---

## Memory rule

```text
getchar() reads one character

putchar(c) prints one character

EOF means no more input

Use int c, not char c, because c must be able to store EOF

'\n' is newline

'\t' is tab

' ' is space

while ((c = getchar()) != EOF) means keep reading characters until input ends
```

Chapter 1 uses `while` here, but the full explanation of loops belongs later in Chapter 3.
