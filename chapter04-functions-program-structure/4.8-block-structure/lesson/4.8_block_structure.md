# 4.8 - Block Structure

## Big idea

A block is code inside braces:

```c
{
    statements
}
```

In C, variables can be declared inside blocks.

That variable only exists inside that block.

---

## C is not fully block-structured

K&R says C is not block-structured like some other languages because functions cannot be defined inside other functions.

This is not allowed:

```c
int main() {
    int helper() {
        return 10;
    }

    return 0;
}
```

In C, functions must be separate.

Correct:

```c
int helper(void) {
    return 10;
}

int main() {
    return 0;
}
```

---

## Variables can have block scope

Even though functions cannot be nested, variables can be declared inside blocks.

Example:

```c
int main() {
    int x = 10;

    if (x > 0) {
        int y = 20;
        printf("%d\n", y);
    }

    return 0;
}
```

Here:

```text
x can be used inside main

y can only be used inside the if block
```

This would be wrong:

```c
int main() {
    int x = 10;

    if (x > 0) {
        int y = 20;
    }

    printf("%d\n", y);

    return 0;
}
```

`y` does not exist outside the block.

---

## Inner variable can hide outer variable

A variable inside a block can have the same name as an outside variable.

Example:

```c
int x = 100;

int main() {
    int x = 10;

    {
        int x = 5;
        printf("inner x = %d\n", x);
    }

    printf("main x = %d\n", x);

    return 0;
}
```

Output:

```text
inner x = 5
main x = 10
```

The inner `x` hides the outer `x` only inside the block.

---

## Scope rule

```text
A variable can be used only from where it is declared to the end of its block.
```

Example:

```c
{
    int a = 5;

    printf("%d\n", a);
}
```

`a` exists only inside that pair of braces.

---

## Why block structure matters

Block structure helps keep variables limited to where they are needed.

This makes code safer and easier to understand.

```text
small block = small variable lifetime
large block = variable visible longer
```

---

## What I need to remember

```text
block = code inside { }

variables declared inside a block only exist inside that block

inner variables can hide outer variables with the same name

functions cannot be defined inside other functions in C

scope means where a name can be used

a variable's scope lasts until the end of its block
```
