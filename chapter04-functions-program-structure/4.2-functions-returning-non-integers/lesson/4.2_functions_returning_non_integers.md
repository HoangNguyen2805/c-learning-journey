# 4.2 - Functions Returning Non-integers

## Big idea

A function does not have to return `int`.

A function can return other types, such as:

```text id="jeoz4g"
double
float
char
void
```

Example:

```c id="5a6m3j"
double average(double a, double b) {
    return (a + b) / 2.0;
}
```

This function returns a `double`.

---

## Return type

The return type comes before the function name.

```c id="w5bj2h"
double average(double a, double b)
```

Meaning:

```text id="u64s33"
average is a function
it takes two double values
it returns a double value
```

---

## Function prototype must match

If a function returns a non-int type, the prototype is very important.

Prototype:

```c id="9du2qa"
double average(double a, double b);
```

Definition:

```c id="g2qirg"
double average(double a, double b) {
    return (a + b) / 2.0;
}
```

The return type must match:

```text id="vss1dh"
prototype return type = double
function definition return type = double
```

---

## Function returning double

Example:

```c id="yd0b4x"
double celsiusToFahrenheit(double celsius) {
    return celsius * 9.0 / 5.0 + 32.0;
}
```

This returns a decimal value.

---

## Function returning char

A function can return one character.

Example:

```c id="xzsz5y"
char getLetterGrade(int score) {
    if (score >= 90)
        return 'A';
    else if (score >= 80)
        return 'B';
    else if (score >= 70)
        return 'C';
    else
        return 'F';
}
```

This function returns a `char`.

---

## Return expression conversion

The value after `return` is converted to the function return type.

Example:

```c id="3toc24"
double half(int x) {
    return x / 2.0;
}
```

The function returns `double`.

But be careful:

```c id="r1yndc"
int half(double x) {
    return x / 2.0;
}
```

This returns `int`, so the decimal part can be lost.

---

## K&R note about `atof`

K&R uses an example function called `atof`.

```text id="rhn8ko"
atof = ASCII to floating-point
```

It converts a string like:

```text id="n4ho21"
"123.45"
```

into a `double` value.

The important idea in this section is not to fully master `atof` yet.

The important idea is:

```text id="9fqm0f"
If a function returns double, C must know that before the function is called.
```

So K&R shows a declaration like:

```c id="g5ba3h"
double atof(char s[]);
```

This tells C:

```text id="xn22eh"
atof takes a character array
atof returns double
```

---

## Old C warning

Older C assumed a function returned `int` if C did not know the function type.

Modern C does not want you to rely on that.

Beginner rule:

```text id="405mil"
Always write function prototypes before main.
```

```text id="ss58wc"

double function returns decimal value

char function returns one character
```
