# 4.1 - Basics of Functions

## Big idea

A function is a named block of code that does a job.

Example:

```c
int add(int a, int b) {
    return a + b;
}
```

Meaning:

```text
int = return type
add = function name
int a, int b = parameters
return a + b = value sent back
```

---
---

## Function definition

A function definition has this form:

```c
return_type function_name(parameter_list) {
    declarations
    statements
}
```

Example:

```c
int square(int x) {
    return x * x;
}
```

This creates a function named `square`.

---

## Function call

A function call uses the function.

```c
square(5)
```

This sends `5` into the function.

Example:

```c
int result = square(5);
```

Now `result` gets the value returned by `square`.

---

## Parameters and arguments

Parameters are the names inside the function definition.

```c
int square(int x)
```

`x` is a parameter.

Arguments are the real values sent during the function call.

```c
square(5)
```

`5` is an argument.

Simple rule:

```text
parameter = variable in the function
argument = value sent into the function
```

---

## Return value

A function can send a value back using `return`.

```c
return x * x;
```

The return type tells what kind of value comes back.

```c
int square(int x)
```

This means `square` returns an `int`.

---

## Void function

A function that does not return a value uses `void`.

```c
void printMessage(void) {
    printf("Hello\n");
}
```

Meaning:

```text
void before function name = returns nothing
void inside parentheses = takes no parameters
```

---

## Function prototype

A prototype tells C about a function before it is used.

```c
int square(int x);
```

This tells C:

```text
square is a function
it takes one int
it returns int
```

Prototype first:

```c
int square(int x);

int main() {
    printf("%d\n", square(5));
    return 0;
}

int square(int x) {
    return x * x;
}
```

---

## Function order

Functions can be written after `main` if there is a prototype before `main`.

Good:

```c
int square(int x);

int main() {
    square(5);
}

int square(int x) {
    return x * x;
}
```

Without the prototype, C may not know the function exists yet.

---

## Functions are not nested

In C, do not define one function inside another function.

Wrong:

```c
int main() {
    int square(int x) {
        return x * x;
    }
}
```

Correct:

```c
int square(int x) {
    return x * x;
}

int main() {
    return 0;
}
```

---

## Call by value

C passes arguments by value.

That means the function gets a copy.

Example:

```c
void change(int x) {
    x = 100;
}
```

If you call:

```c
int number = 5;
change(number);
```

`number` is still `5` after the function call.

The function changed only its copy.

## K&R note

K&R 4.1 uses functions to split a program into smaller pieces.

Main idea:

```text
main should control the program
helper functions should do smaller jobs
```

This makes code easier to read, test, and reuse.
