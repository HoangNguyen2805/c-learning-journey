# 4.5 - Header Files

## Big idea

A header file is a `.h` file that stores declarations shared by other `.c` files.

Header files often contain:

```text id="9iu4om"
function prototypes
#define constants
extern declarations
type declarations later
```

Example:

```c id="4qo9gk"
int add(int a, int b);
```

This prototype can go in a header file.

---

## Why header files exist

When a program gets bigger, it may have many `.c` files.

Example:

```text id="v5t88m"
main.c
math_utils.c
math_utils.h
```

`main.c` wants to use functions from `math_utils.c`.

The header file tells `main.c` what functions exist.

---

## Header file example

Header file:

```c id="wm7wea"
int add(int a, int b);
int multiply(int a, int b);
```

This does not create the functions.

It only announces them.

The real function definitions go in a `.c` file.

---

## `.h` file vs `.c` file

Header file `.h`:

```text id="2zzq40"
announces functions
stores shared declarations
gets included with #include
```

Source file `.c`:

```text id="cv08l3"
contains real function definitions
contains executable code
gets compiled
```

---

## Include your own header file

Use double quotes for your own header files.

```c id="95u9lt"
#include "math_utils.h"
```

Use angle brackets for standard library headers.

```c id="gxbiil"
#include <stdio.h>
```

Simple rule:

```text id="n1ny4u"
<stdio.h> = system/header from C library
"math_utils.h" = your own header file
```

---

## Function prototype in header

Instead of putting prototypes directly above `main`:

```c id="x8tg3w"
int add(int a, int b);
int multiply(int a, int b);
```

You can put them in:

```text id="g9xvck"
math_utils.h
```

Then include the header:

```c id="kn3f7v"
#include "math_utils.h"
```

---

## Do not put normal function definitions in headers

Usually, do not put full function bodies in a header file.

Avoid this in `.h`:

```c id="bwu5q9"
int add(int a, int b) {
    return a + b;
}
```

Better:

```text id="1w8ybu"
prototype in .h
definition in .c
```

---

## External variables in headers

If an external variable is defined in one `.c` file, the header should use `extern`.

Header:

```c id="x7dmmg"
extern int count;
```

Source file:

```c id="wrfax4"
int count;
```

Simple rule:

```text id="k04f3z"
.h file announces external variable with extern
.c file defines the real variable
```

---

## Header guards

A header guard prevents a header file from being included more than once.

Example:

```c id="z78vj2"
#ifndef MATH_UTILS_H
#define MATH_UTILS_H

int add(int a, int b);
int multiply(int a, int b);

#endif
```

Meaning:

```text id="142lsj"
if MATH_UTILS_H is not defined
define it
include the header content
end the check
```

Beginner rule:

```text id="yq3kh2"
Put header guards in your own .h files.
```

---

## How to compile multiple `.c` files

If the program uses more than one `.c` file, compile them together.

Example:

```bash id="h8qdmg"
gcc main.c math_utils.c -o program
```

The `.h` file is included by the `.c` files, but you normally do not list the `.h` file in the gcc command.

---

## What I need to remember

```text id="72ijqq"
header file = .h file

header files store shared declarations

function prototypes can go in .h files

.c files contain real function definitions

#include "file.h" includes your own header file

#include <stdio.h> includes a standard library header

use header guards in your own .h files

extern declarations can go in headers

compile multiple .c files together

do not put normal function bodies in headers
```
