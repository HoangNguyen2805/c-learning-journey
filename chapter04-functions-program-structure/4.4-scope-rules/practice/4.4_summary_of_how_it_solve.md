# 4.4 Practice Summary - Fibonacci with Scope (How We Solved It)

## The Goal
Generate the first N Fibonacci numbers AND keep a running external counter
of how many were generated - using THREE different variables all named "n",
each living in a different scope, without letting them interfere.

The three "n" variables:
1. EXTERNAL n = 0           -> tracks total numbers generated (shared across program)
2. main()'s LOCAL n         -> stores the user's input (how many to generate)
3. fib()'s PARAMETER n      -> which position to calculate, changes every call

---

## Step 1 - Build main()'s counting loop (no math yet)

main() doesn't calculate anything itself. Its only job: ask the user for
input, then loop that many times.

```c
int main() {
    int n;   // LOCAL n - the user's input
    scanf("%d", &n);

    int i;
    for (i = 1; i <= n; i++) {
        // will call fib(i) here once fib() exists
    }
}
```

Key idea: main()'s loop counter is `i`, NOT `n`. `n` only sets the LIMIT
(`i <= n`); `i` is what actually changes and gets passed to fib().

---

## Step 2 - Build fib() as its own self-contained calculator

fib() knows nothing about main()'s n, or the external n. It only knows
its OWN parameter n (the position it's been asked to calculate).

```c
int fib(int n) {
    int a = 1;   // position 1
    int b = 1;   // position 2

    if (n == 1 || n == 2) {
        return 1;   // already known, no loop needed
    }

    int i;
    int next;
    for (i = 3; i <= n; i++) {
        next = a + b;   // calculate BEFORE overwriting anything
        a = b;           // slide a forward
        b = next;        // slide b forward
    }

    return b;   // b always holds the most recently calculated number
}
```

Key idea: by the time the loop ends, b is sitting exactly on the answer
for position n. a is always one step behind b, so we return b, not a.

---

## Step 3 - Connect main()'s loop to fib(), print each result

```c
for (i = 1; i <= n; i++) {
    printf("%d\n", fib(i));   // fib(i) - pass the CHANGING i, not n
}
```

Key idea: each round passes a DIFFERENT number (i = 1, 2, 3...) into
fib()'s separate parameter n. fib(i) returns one answer per call, and
printf displays whatever fib() returned.

---

## Step 4 - The external counter problem (the real trap)

Goal: count how many numbers were printed, using the EXTERNAL n.

THE TRAP: main() already has its own LOCAL n. Because of that, main()
can NEVER see the external n directly - not to increment it, not even
to print it. Every use of "n" inside main() means the local one, period.

SOLUTION: write separate helper functions that have NO local n of their
own - only then can they "see" the external n.

```c
void incrementCounter(void) {
    n = n + 1;   // no local n here -> reaches the EXTERNAL n
}

void showCount(void) {
    printf("Total Fibonacci numbers generated: %d\n", n);  // same trick
}
```

Call incrementCounter() once per loop iteration (inside the loop), and
call showCount() once, after the loop ends, to report the final total:

```c
for (i = 1; i <= n; i++) {
    printf("%d\n", fib(i));
    incrementCounter();
}

showCount();
```

---

## The Big Lesson

Once a function declares its OWN local variable with a name, that
function can NEVER reach an external variable with the same name -
not even to read it, not even to print it. The only way to touch the
external one is from a DIFFERENT function that doesn't redeclare that
name locally.

This is why main()'s local n and the external n - despite sharing a
name and even ending up with the same VALUE by coincidence in this
test (both = 6) - are completely separate variables the entire time.
