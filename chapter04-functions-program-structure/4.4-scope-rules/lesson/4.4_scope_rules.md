# 4.4 - Scope Rules

## Big idea

Scope means where a name can be used in a program.

A name can be:

```text
variable name
function name
parameter name
external variable name
```

Scope answers this question:

```text
Where is this name visible?
```

---

## Local variables

A local variable is declared inside a function.

```c
void function(void) {
    int x = 10;
}
```

`x` can only be used inside that function. Another function cannot use `x`.

---

## Function parameters

Function parameters are also local to the function.

```c
int square(int x) {
    return x * x;
}
```

`x` only exists inside `square`. The `x` in `square` is separate from any variable named `x` in `main`.

---

## External variables

An external variable is declared outside all functions.

```c
int count;

void addOne(void) {
    count++;
}
```

External variables can be used by multiple functions.

---

## Definition vs declaration

A **definition** creates the variable and storage.

```c
int count;
```

A **declaration** (using `extern`) tells C the variable exists somewhere else — it does NOT create a new variable.

```c
extern int count;
```

```text
definition       = creates the variable
extern declaration = announces the variable
```

---

## `extern`

```c
extern int count;
```

This tells C: "trust me, `count` is defined somewhere — don't create a new one, just use the existing one."

**When do you actually need `extern`?**

If the external variable is defined *before* a function that uses it, no `extern` is needed:

```c
int count;

void addOne(void) {
    count++;   // fine - count already exists above
}
```

If the variable is defined *later* in the file, `extern` is needed first:

```c
extern int count;

void addOne(void) {
    count++;   // works because of the extern promise above
}

int count;   // actual definition, later
```

**The bigger real-world reason `extern` exists:** in larger programs split across multiple files, `extern` is how one file says "use the variable that's defined in a different file." You'll see this more once programs grow beyond a single file.

---

## Scope of external names

For external variables and functions, scope begins where the name is declared and continues to the end of the file. Order matters — if C hasn't seen the name yet, it doesn't know what it is.

---

## Same name in different scopes (shadowing)

A local variable can hide an external variable with the same name.

```c
int value = 100;

void show(void) {
    int value = 5;
    printf("%d\n", value);   // prints 5 - the LOCAL value
}
```

Inside `show`, `value` refers to the local variable. The external `value` still exists, but it's hidden for as long as you're inside `show`.

If you remove the local declaration entirely:
```c
int value = 100;

void show(void) {
    printf("%d\n", value);   // prints 100 - now it reaches the external value
}
```

---

## The real bug this causes

This is the most common mistake scope rules lead to — accidentally creating a new local variable instead of modifying the external one:

```c
int total = 0;

void addToTotal(int amount) {
    int total = amount;   // BUG: this creates a NEW local total.
                          // It does NOT touch the external total.
}

int main() {
    addToTotal(50);
    printf("%d\n", total);   // prints 0, NOT 50
}
```

The line `int total = amount;` looks like it updates the external `total`, but the `int` keyword secretly declares a brand new local variable that shadows it. The external `total` is never touched.

**The fix:** drop the `int` when you mean to use the existing external variable:
```c
void addToTotal(int amount) {
    total = amount;   // no "int" - this correctly modifies the external total
}
```

Beginner rule:

```text
If you mean to use the external variable, do NOT re-declare it locally
with the same name. Just assign to it directly, without "int" in front.
```

---

## Block scope

A variable declared inside `{ }` belongs only to that block.

```c
{
    int temp = 10;
}
```

`temp` cannot be used outside that block. (Covered more in 4.8 — just be aware it's part of the same scope idea.)

---

## Why scope matters

Scope prevents name conflicts and makes clear which functions can access which variables.

```text
Use local variables first.
Use parameters and return values when possible.
Use external variables only when shared state is genuinely needed.
```
