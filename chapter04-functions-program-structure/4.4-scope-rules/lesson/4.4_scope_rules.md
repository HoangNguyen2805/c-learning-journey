# 4.4 - Scope Rules

## Big idea

Scope means where a name can be used in a program.

A name can be:

```text
variable name
function name
parameter name
external variable name
```

Scope answers this question:

```text
Where is this name visible?
```

---

## Local variables

A local variable is declared inside a function.

```c
void function(void) {
    int x = 10;
}
```

`x` can only be used inside that function.

Another function cannot use `x`.

---

## Function parameters

Function parameters are also local to the function.

```c
int square(int x) {
    return x * x;
}
```

`x` only exists inside `square`.

The `x` in `square` is separate from any variable named `x` in `main`.

---

## External variables

An external variable is declared outside all functions.

```c
int count;
```

External variables can be used by multiple functions.

```c
int count;

void addOne(void) {
    count++;
}
```

---

## Definition vs declaration

This is important in K&R 4.4.

A definition creates the variable and storage.

```c
int count;
```

A declaration tells C that the variable exists somewhere.

```c
extern int count;
```

Simple rule:

```text
definition = creates the variable
extern declaration = announces the variable
```

---

## `extern`

`extern` tells C:

```text
This variable is defined somewhere else.
```

Example:

```c
extern int count;
```

This does not create a new `count`.

It only tells C that `count` exists.

---

## Why `extern` is needed

If an external variable is defined before a function uses it, no `extern` is needed.

```c
int count;

void addOne(void) {
    count++;
}
```

But if the variable is defined later, C needs an `extern` declaration first.

```c
extern int count;

void addOne(void) {
    count++;
}

int count;
```

---

## Scope of external names

For external variables and functions, scope usually begins where the name is declared and continues to the end of the file.

That means order matters.

If C has not seen the name yet, it does not know what it is.

---

## Same name in different scopes

A local variable can hide an external variable with the same name.

```c
int value = 100;

void show(void) {
    int value = 5;
    printf("%d\n", value);
}
```

Inside `show`, `value` means the local variable.

The external `value` is hidden inside that function.

Beginner rule:

```text
Avoid using the same name for local and external variables.
```

---

## Block scope

A variable declared inside `{ }` belongs only to that block.

```c
{
    int temp = 10;
}
```

`temp` cannot be used outside that block.

Block scope is covered more later in 4.8, but it is part of understanding scope.

---

## Why scope matters

Scope helps prevent name conflicts.

It also helps you know which functions can access which variables.

Good code usually keeps variables as local as possible.

Beginner rule:

```text
Use local variables first.
Use parameters and return values when possible.
Use external variables only when shared state is needed.
```

---

## What I need to remember

```text
scope = where a name can be used

local variable = visible inside one function

parameter = local to its function

external variable = outside all functions

definition = creates storage

extern declaration = announces a variable defined somewhere else

external names are visible from declaration to end of file

local names can hide external names

block variables only exist inside their block

keep variables as local as possible
```
