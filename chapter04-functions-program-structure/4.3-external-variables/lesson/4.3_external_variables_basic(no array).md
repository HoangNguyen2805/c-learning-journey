# 4.3 - External Variables (Basic — No Arrays)

## Big idea

An external variable is a variable declared outside all functions.

Example:

```c
int count;

void increment(void) {
    count = count + 1;
}

int main() {
    count = 0;
    increment();
    increment();
    printf("%d\n", count);   // prints 2
    return 0;
}
```

Because `count` is declared outside all functions, both `main()` and
`increment()` can read and change the same `count` — not separate copies.

---

## Local variable vs external variable

Local variable — belongs only to one function:

```c
void function(void) {
    int x;   // only function() can see this x
}
```

External variable — shared across functions in the same file:

```c
int x;   // declared OUTSIDE any function

void function(void) {
    x = x + 1;   // this is THE SAME x as everywhere else
}
```

---

## Why external variables exist

Normally, a function only knows about:
- its own parameters
- its own local variables

If two functions need to work with the *same* piece of data — without
passing it back and forth through parameters and return values every
time — an external variable lets them both reach it directly.

---

## A simple shared-counter example

```c
#include <stdio.h>

int total = 0;   // external variable - shared

void addToTotal(int amount) {
    total = total + amount;
}

void resetTotal(void) {
    total = 0;
}

int main() {
    addToTotal(10);
    addToTotal(5);
    printf("Total: %d\n", total);   // 15

    resetTotal();
    printf("Total after reset: %d\n", total);   // 0

    return 0;
}
```

`addToTotal` and `resetTotal` both reach the SAME `total` — neither one
receives it as a parameter, and neither one returns it. They just use it
directly because it's declared outside all functions.

---

## External variable warning

External variables can make programs harder to understand if overused —
because it's not obvious from reading a function's parameter list that
it depends on (or changes) outside data.

Beginner rule:

```text
Use local variables first.
Use parameters and return values when possible.
Use external variables only when functions truly need shared state.
```
