## K&R's real-world motivation

K&R introduces external variables using a calculator stack — a case where
TWO functions (`push` and `pop`) both need access to the SAME stack data.

```c
#define MAXVAL 100

int sp = 0;            // next free stack position
double val[MAXVAL];    // array holding stack values

void push(double f) {
    val[sp] = f;
    sp++;
}

double pop(void) {
    sp--;
    return val[sp];
}
```

## Stack idea (this part doesn't need arrays to understand)

A stack is last-in, first-out:

```text
push 10
push 20
pop gives 20   <- last one in, first one out
pop gives 10
```

## Why sp and val are external

Both `push()` and `pop()` need to read and modify the SAME `sp` and the
SAME `val` array. If they were local variables instead, each function
would get its own private, useless copy — push() would modify a stack
that pop() could never see.

This is the clearest real example of WHY external variables matter:
some problems genuinely require multiple functions to share live data.
