# 4.3 - External Variables

## Big idea

An external variable is a variable declared outside all functions.

Example:

```c
int count;
```

Because it is outside all functions, more than one function can use it.

---

## Local variable vs external variable

Local variable:

```c
void function(void) {
    int x;
}
```

`x` belongs only to that function.

External variable:

```c
int x;

void function(void) {
    x = x + 1;
}
```

`x` can be shared by multiple functions in the same file.

---

## Why external variables exist

External variables are useful when several functions need to share the same data.

K&R uses this idea for a calculator stack.

The shared data is:

```text
val[] = stack storage
sp = stack position
```

Both `push()` and `pop()` need the same `val[]` and `sp`.

---

## Stack idea

A stack is last in, first out.

```text
push = put value on top
pop = take value from top
```

Example:

```text
push 10
push 20
pop gives 20
pop gives 10
```

The last value pushed is the first value popped.

---

## K&R stack variables

K&R uses external variables like this:

```c
#define MAXVAL 100

int sp = 0;
double val[MAXVAL];
```

Meaning:

```text
MAXVAL = maximum stack size
sp = next free stack position
val = array holding stack values
```

---

## Push

`push` adds a value to the stack.

```c
void push(double f) {
    val[sp] = f;
    sp++;
}
```

Meaning:

```text
store f at val[sp]
move sp to the next position
```

---

## Pop

`pop` removes and returns the top value.

```c
double pop(void) {
    sp--;
    return val[sp];
}
```

Meaning:

```text
move sp back
return the value at val[sp]
```

---

## Why `sp` and `val` are external

`push()` needs `sp` and `val`.

`pop()` also needs `sp` and `val`.

So K&R puts them outside all functions.

That way both functions share the same stack.

---

## External variable warning

External variables can make programs harder to understand if overused.

Beginner rule:

```text
Use local variables first.
Use parameters and return values when possible.
Use external variables only when functions truly need shared state.
```

---

## What I need to remember

```text
external variable = variable outside all functions

external variables can be shared by multiple functions

local variables belong only to one function

K&R uses external variables for a stack

val[] stores stack values

sp stores the next stack position

push adds a value

pop removes and returns a value

external variables are useful but should not be overused
```
