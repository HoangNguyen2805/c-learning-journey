# 4.11 - The C Preprocessor

## Big idea

The preprocessor runs **before** the C compiler and processes directives (commands starting with `#`).

Think of it as a text substitution tool: it finds and replaces text, includes files, and conditionally includes/excludes code — all before the compiler ever sees the actual C code.

Examples of preprocessor directives:

```c
#include <stdio.h>           // include a file
#define MAX 100              // substitute MAX with 100
#ifndef STUDENT_UTILS_H      // if not defined
#define STUDENT_UTILS_H      // define it
#endif                       // end the condition
```

---

## What the preprocessor does

The preprocessor can:

- Include files into your program
- Replace symbolic names with their values
- Create function-like macros (text substitution)
- Conditionally include/exclude code based on definitions
- Undefine macros
- Handle conditional directives (`#if`, `#ifdef`, `#ifndef`, `#elif`, `#else`)

K&R 4.11 focuses on:

- File inclusion (`#include`)
- Macro substitution (`#define`)
- Conditional inclusion (`#ifdef`, `#ifndef`, `#endif`)

---

# 1. File inclusion

`#include` copies another file into your program before compiling.

### Standard library header:

```c
#include <stdio.h>
```

The `< >` syntax tells the preprocessor: "look in the standard library directories."

### Your own header file:

```c
#include "student_utils.h"
```

The `" "` syntax tells the preprocessor: "look in the current directory (or project directory) first."

### Rule of thumb:

```text
< > = standard library headers (stdio.h, stdlib.h, string.h, math.h, etc.)
" " = your own header files
```

---

# 2. Macro substitution with #define

`#define` creates a symbolic name that the preprocessor replaces with a value everywhere in the code.

### Simple constant macro:

```c
#define MAXLINE 1000
#define PI 3.14159
#define PASSING_SCORE 70
```

After these definitions, every occurrence of `MAXLINE` is replaced with `1000`, `PI` with `3.14159`, etc.

### How it works:

```c
char line[MAXLINE];  // becomes: char line[1000];
double area = PI * radius * radius;  // becomes: double area = 3.14159 * radius * radius;
```

### Important syntax rules:

```text
✓ Correct:   #define MAX 100
✗ Wrong:     #define MAX = 100
✗ Wrong:     #define MAX = 100;

✓ Correct:   #define PI 3.14159
✗ Wrong:     #define PI = 3.14159;
```

**No `=` or `;` in `#define`** — the preprocessor just does text substitution.

---

# 3. Function-like macros

A macro can look and behave like a function call.

### Basic example:

```c
#define square(x) ((x) * (x))
```

Then when you write:

```c
int result = square(5);
```

The preprocessor expands it to:

```c
int result = ((5) * (5));  // = 25
```

### Why parentheses are critical:

**Bad macro (without parentheses):**

```c
#define square(x) x * x
```

If you call `square(1 + 2)`:

```c
1 + 2 * 1 + 2  // evaluated as: 1 + (2*1) + 2 = 5 ✗ WRONG
```

Due to operator precedence, you get the wrong answer.

**Good macro (with parentheses):**

```c
#define square(x) ((x) * (x))
```

If you call `square(1 + 2)`:

```c
((1 + 2) * (1 + 2))  // = (3) * (3) = 9 ✓ CORRECT
```

### Rule for macros:

```text
Always wrap parameters in parentheses: ((x) * (x))
Always wrap the entire expression in parentheses: ((x) * (x))
```

### More function-like macro examples:

```c
#define max(a, b) ((a) > (b) ? (a) : (b))
#define min(a, b) ((a) < (b) ? (a) : (b))
#define abs(x) ((x) < 0 ? -(x) : (x))
```

# 4. Macro gotchas (things that can go wrong)

### Problem 1: Side effects in macro arguments

Bad macro usage:

```c
#define increment(x) ((x) + 1)

int i = 5;
int result = increment(i++);
```

This expands to:

```c
int result = ((i++) + 1);
```

`i` gets incremented, but you might not expect it. This is a hidden side effect.

**Worse example:**

```c
#define addScores(a, b) ((a) + (b))

int score1 = 10, score2 = 20;
int total = addScores(score1++, score2++);  // Both score1 and score2 increment!
```

**Best practice:** Use function arguments in macros carefully, and avoid expressions with side effects.

### Problem 2: Type safety (macros don't check types)

Functions check argument types. Macros don't.

```c
// Function (type-safe):
int add(int a, int b) {
    return a + b;
}

// Macro (not type-safe):
#define add(a, b) ((a) + (b))
```

The macro works with integers, floats, doubles, etc. — but also silently with incompatible types, which can hide bugs.

# 5. Conditional inclusion

Conditional inclusion means the preprocessor includes or excludes code based on whether something is defined.

### #ifdef (if defined)

```c
#ifdef DEBUG
    printf("Debug mode is ON\n");
#endif
```

Meaning: if `DEBUG` is defined, include this code. Otherwise, skip it.

### #ifndef (if not defined)

```c
#ifndef RELEASE
    printf("Not a release build\n");
#endif
```

Meaning: if `RELEASE` is NOT defined, include this code.

### Header guard pattern (most common use of #ifndef)

In header files, use this pattern to prevent double inclusion:

```c
#ifndef STUDENT_UTILS_H
#define STUDENT_UTILS_H

/* function prototypes go here */
int addScores(int a, int b);
int calculateAverage(int total, int count);

#endif
```

**Why?** If you `#include` the same header file multiple times (directly or indirectly through other headers), the header guard prevents redefinition errors.

---

# 6. #if, #elif, #else

More powerful conditional directives:

### #if with expressions:

```c
#if DEBUG_LEVEL > 2
    printf("Deep debugging enabled\n");
#endif
```

### #if, #elif, #else chain:

```c
#define VERSION 2

#if VERSION == 1
    printf("Using version 1 API\n");
#elif VERSION == 2
    printf("Using version 2 API\n");
#else
    printf("Unknown version\n");
#endif
```

### Combining conditions:

```c
#if defined(DEBUG) && !defined(PRODUCTION)
    printf("Debug mode, not production\n");
#endif
```

---

# 7. #undef (undefine a macro)

Removes a macro definition.

```c
#define MAX 100
printf("%d\n", MAX);  // prints 100

#undef MAX
// MAX is no longer defined

printf("%d\n", MAX);  // ERROR: MAX is undefined
```
## #endif - Closing a preprocessor conditional

`#endif` (end if) closes a preprocessor conditional block that started with `#ifdef`, `#ifndef`, or `#if`.

### Pattern:

```c
#ifdef DEBUG
    // code here only if DEBUG is defined
#endif
```

`#ifdef` opens the conditional, `#endif` closes it.

---

## DEBUG - A common macro for development/debug mode

`DEBUG` is not a built-in keyword — it's just a **name you choose** to define.

### Usage pattern:

```c
#define DEBUG    // Define DEBUG (it's empty, but it exists)

#ifdef DEBUG
    printf("Debug: variable x = %d\n", x);
#endif
```

If `#define DEBUG` is present, the debug code is included and compiled.
If you remove or comment out `#define DEBUG`, the debug code is excluded.

### Why is this useful?

- **Development:** Keep `#define DEBUG` so you see debug messages
- **Production:** Remove `#define DEBUG` and recompile — debug messages disappear

It's a way to turn debug output on/off without rewriting code.

### Example:

```c
#define DEBUG    // uncomment to enable debug output

int main() {
#ifdef DEBUG
    printf("Program started\n");
#endif
    
    // ... rest of program ...
    
#ifdef DEBUG
    printf("Program ending\n");
#endif
    
    return 0;
}
```

# 8. Predefined macros (compiler-provided)

The C compiler automatically defines certain macros:

```c
__LINE__   // current line number
__FILE__   // current filename (as a string)
__DATE__   // compilation date (as a string)
__TIME__   // compilation time (as a string)
__STDC__   // defined if compiler is standard-compliant
```

### Example:

```c
#include <stdio.h>

int main() {
    printf("File: %s\n", __FILE__);
    printf("Line: %d\n", __LINE__);
    printf("Date: %s\n", __DATE__);
    printf("Time: %s\n", __TIME__);
    return 0;
}
```

Output might be:

```
File: main.c
Line: 5
Date: Jul  4 2026
Time: 14:32:45
```

# What to remember

```text
Preprocessor runs BEFORE the compiler

All preprocessor directives start with #

#include <file>   = include standard library header
#include "file"   = include your own header file

#define NAME VALUE = create a constant or macro (no = or ; needed)

Macros do text substitution — be careful with side effects
Macros don't check types — be careful with mixed types
Always use parentheses in function-like macros: #define square(x) ((x) * (x))

#ifdef NAME = if NAME is defined
#ifndef NAME = if NAME is not defined
#if EXPRESSION = if expression is true
#elif, #else = conditional chains
#endif = end a conditional block

Header guards prevent double inclusion:
  #ifndef FILE_NAME_H
  #define FILE_NAME_H
  ...
  #endif

#undef = remove a macro definition

Predefined macros: __LINE__, __FILE__, __DATE__, __TIME__, __STDC__

4.11 teaches the # commands and how to use them safely
```

## Complete working examples

### Example 1: Constants and basic macros

```c
#include <stdio.h>

#define MAX_SCORE 100
#define PASSING_SCORE 70
#define GRADE_A 90
#define GRADE_B 80
#define GRADE_C 70

int main() {
    int score = 85;

    printf("Max score: %d\n", MAX_SCORE);
    printf("Passing score: %d\n", PASSING_SCORE);
    printf("Student score: %d\n", score);

    if (score >= GRADE_A) {
        printf("Grade: A\n");
    } else if (score >= GRADE_B) {
        printf("Grade: B\n");
    } else if (score >= GRADE_C) {
        printf("Grade: C\n");
    } else {
        printf("Grade: F\n");
    }

    return 0;
}
```

### Example 2: Function-like macros with proper parentheses

```c
#include <stdio.h>

#define square(x) ((x) * (x))
#define cube(x) ((x) * (x) * (x))
#define max(a, b) ((a) > (b) ? (a) : (b))
#define min(a, b) ((a) < (b) ? (a) : (b))
#define abs(x) ((x) < 0 ? -(x) : (x))

int main() {
    int number = 5;
    double value = 2.5;

    printf("Square of %d: %d\n", number, square(number));
    printf("Cube of %d: %d\n", number, cube(number));
    printf("max(3, 7): %d\n", max(3, 7));
    printf("min(3, 7): %d\n", min(3, 7));
    printf("abs(-10): %d\n", abs(-10));
    
    // Safe with expressions because of parentheses:
    printf("square(1 + 2): %d\n", square(1 + 2));  // = 9 ✓

    return 0;
}
```

### Example 3: Conditional compilation

```c
#include <stdio.h>

#define DEBUG
#undef RELEASE

int main() {
    printf("Program started\n");

#ifdef DEBUG
    printf("[DEBUG] Debug mode is ON\n");
#endif

#ifndef RELEASE
    printf("[INFO] This is NOT a release build\n");
#endif

#ifdef PRODUCTION
    printf("Running in production\n");
#else
    printf("Running in development mode\n");
#endif

    printf("Program ended\n");

    return 0;
}
```

### Example 4: Header guard pattern

**File: math_utils.h**

```c
#ifndef MATH_UTILS_H
#define MATH_UTILS_H

// Prototypes
int add(int a, int b);
int subtract(int a, int b);
int multiply(int a, int b);

#endif
```

**File: main.c**

```c
#include <stdio.h>
#include "math_utils.h"

int add(int a, int b) {
    return a + b;
}

int subtract(int a, int b) {
    return a - b;
}

int multiply(int a, int b) {
    return a * b;
}

int main() {
    printf("5 + 3 = %d\n", add(5, 3));
    printf("5 - 3 = %d\n", subtract(5, 3));
    printf("5 * 3 = %d\n", multiply(5, 3));
    return 0;
}
```

## Summary

The preprocessor is a powerful tool for:
- Reusing code via `#include`
- Creating readable constants and macros with `#define`
- Building flexible, configurable code with conditional compilation

Master the basics (`#include`, simple `#define`, `#ifdef`/`#ifndef`), and you'll have all the tools you need for most C programs. Use function-like macros carefully (with parentheses), and prefer actual functions when type safety matters.
