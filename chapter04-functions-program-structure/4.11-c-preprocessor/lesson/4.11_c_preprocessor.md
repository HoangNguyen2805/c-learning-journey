# 4.11 - The C Preprocessor

## Big idea

The preprocessor runs before the C compiler.

Preprocessor commands start with `#`.

Examples:

```c
#include <stdio.h>
#define MAX 100
#ifndef STUDENT_UTILS_H
#define STUDENT_UTILS_H
#endif
```

These are called **preprocessor directives**.

---

## What the preprocessor does

The preprocessor can:

```text
include files
replace names with values
create macros
compile code only if a condition is true
```

K&R 4.11 focuses on:

```text
file inclusion
macro substitution
conditional inclusion
```

---

# 1. File inclusion

`#include` copies another file into your program before compiling.

Standard library header:

```c
#include <stdio.h>
```

Your own header file:

```c
#include "student_utils.h"
```

Rule:

```text
< > = standard library header

" " = your own header file
```

---

# 2. Macro substitution with #define

`#define` creates a symbolic name.

Example:

```c
#define MAXLINE 1000
```

After this, every time the preprocessor sees `MAXLINE`, it replaces it with `1000`.

Example:

```c
char line[MAXLINE];
```

becomes like:

```c
char line[1000];
```

Important:

```text
#define does not use =
#define does not use ;
```

Correct:

```c
#define MAX 100
```

Wrong:

```c
#define MAX = 100;
```

---

# 3. Function-like macros

A macro can also look like a function.

Example:

```c
#define square(x) ((x) * (x))
```

Then:

```c
square(5)
```

becomes:

```c
((5) * (5))
```

Result:

```text
25
```

---

## Why macro parentheses matter

Bad macro:

```c
#define square(x) x * x
```

If you write:

```c
square(1 + 2)
```

It becomes:

```c
1 + 2 * 1 + 2
```

That gives the wrong result because multiplication happens first.

Better macro:

```c
#define square(x) ((x) * (x))
```

Then:

```c
square(1 + 2)
```

becomes:

```c
((1 + 2) * (1 + 2))
```

Result:

```text
9
```

---

# 4. Conditional inclusion

Conditional inclusion means the preprocessor includes code only if a condition is true.

Example:

```c
#ifdef DEBUG
    printf("Debug mode\n");
#endif
```

Meaning:

```text
if DEBUG is defined,
include this code
```

---

## #ifndef

`#ifndef` means:

```text
if not defined
```

Example:

```c
#ifndef STUDENT_UTILS_H
#define STUDENT_UTILS_H

int addScores(int a, int b);

#endif
```

This is a header guard.

It prevents the same header file from being included more than once.

---

## Header guard pattern

Use this pattern in header files:

```c
#ifndef FILE_NAME_H
#define FILE_NAME_H

/* prototypes go here */

#endif
```

Example:

```c
#ifndef CALC_H
#define CALC_H

void push(double);
double pop(void);

#endif
```

---

## #undef

`#undef` removes a macro definition.

Example:

```c
#define MAX 100
#undef MAX
```

After `#undef`, `MAX` is no longer defined.

---

## What I need to remember

```text
preprocessor runs before compiler

preprocessor directives start with #

#include copies another file into the program

#define creates symbolic constants or macros

#define does not use = or ;

function-like macros should use parentheses

#ifdef checks if something is defined

#ifndef checks if something is not defined

#endif ends a preprocessor condition

header guards prevent double include

4.11 teaches the # commands deeper
```
