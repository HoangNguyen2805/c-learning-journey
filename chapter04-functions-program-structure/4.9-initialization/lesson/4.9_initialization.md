# 4.9 - Initialization

## Big idea

Initialization means giving a variable its starting value at the moment
it is created.

```c
int x = 10;   // create x AND start it with 10
```

---

## The three rules (the heart of this section)

```text
automatic local variable without initialization = GARBAGE (unknown value)
external variable without initialization        = 0 (automatic)
static variable without initialization          = 0 (automatic)
```

Everything else in this section builds on these three rules.

---

## Rule 1: Local automatic variables = garbage if uninitialized

```c
int main() {
    int x;              // declared but NOT initialized
    printf("%d\n", x);  // prints GARBAGE - could be any random value
    return 0;
}
```

A local variable without initialization contains whatever leftover bits
happened to be sitting in that memory location. It is NOT guaranteed to
be 0, and it changes unpredictably between runs.

## I already experienced this bug myself

In my Fibonacci practice (4.4), I called enterNumber(a, b) with
uninitialized variables and got garbage output like 1835822505.
That was this exact rule in action - main()'s a and b were never
initialized, and the function couldn't reach them to set values
(call by value), so the math ran on pure garbage.

Lesson learned the hard way:

```text
ALWAYS initialize local variables before using them.
```

---

## Rule 2 & 3: External and static variables = automatic 0

```c
int externalCount;          // automatically starts at 0

void countCalls(void) {
    static int staticCount;  // ALSO automatically starts at 0

    externalCount++;
    staticCount++;
}
```

Both start at 0 without writing "= 0" explicitly.

BUT: writing "= 0" explicitly anyway is still good practice - it makes
your intent obvious to anyone reading the code (including future me).

```c
int externalCount = 0;      // same behavior, clearer intent
```

---

## Static local variables initialize ONLY ONCE

Already proven in my 4.6 practice:

```c
void countCalls(void) {
    static int count = 0;   // this "= 0" runs ONE time ever -
                             // the very first call - never again
    count++;
    printf("%d\n", count);
}
```

Three calls print: 1, 2, 3 (NOT 1, 1, 1)

The initializer does NOT re-run on later calls. If it did, the counter
could never grow. This is the proof I confirmed in quiz 4.6 Q4.

---

## External/static initializers must be CONSTANT values

```c
static int max = 100;        // OK - 100 is a constant
int limit = 50;              // OK - constant

int x = someFunction();      // NOT allowed for external/static -
                              // must be a constant, not a computed value
```

Why: external and static variables are initialized BEFORE the program
even starts running - so their starting values must be known at
compile time, not computed at runtime.

(Local automatic variables CAN be initialized with computed values,
since they're created while the program is running.)

## Array initialization

```c
int numbers[5] = {10, 20, 30, 40, 50};
```

```text
numbers[0] = 10
numbers[1] = 20
numbers[2] = 30
numbers[3] = 40
numbers[4] = 50
```

## Partial array initialization - missing values become 0

```c
int numbers[5] = {10, 20};
```

```text
numbers[0] = 10
numbers[1] = 20
numbers[2] = 0     <- filled automatically
numbers[3] = 0     <- filled automatically
numbers[4] = 0     <- filled automatically
```

## Character array / string initialization

```c
char word[] = {'h', 'i', '\0'};   // manual version
char word[] = "hi";               // string literal version - same thing
```

The string literal version automatically adds the '\0' at the end.

```text
"hi" = 'h', 'i', '\0'
```

## What I need to remember

```text
initialization = giving a variable a starting value when created

local automatic without init = GARBAGE (I hit this bug in Fibonacci 4.4)

external without init = 0 automatically

static without init = 0 automatically

static local initializer runs ONLY ONCE (proven in 4.6 practice)

external/static initializers must be CONSTANTS (set before program runs)

always initialize local variables explicitly - garbage bugs are real

[deferred] arrays: missing initializer values become 0
```
