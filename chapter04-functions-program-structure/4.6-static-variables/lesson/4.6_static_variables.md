# 4.6 - Static Variables

## Big idea

K&R 4.6 uses `static` mostly to make variables **private to one file**.

The book is thinking about the calculator program:

```text
calc.h
main.c
getop.c
stack.c
getch.c
```

Some files need private data.

Other files should not directly touch that private data.

---

## The main rule

```text
static outside a function = private to this .c file
```

Example:

```c
static int sp = 0;
```

This means:

```text
sp can only be used inside this file.
Other .c files cannot use sp directly.
```

---

## External name vs internal name

### External name

An external name can be used by other files.

Example from calculator:

```c
void push(double f);
double pop(void);
```

These functions are shared through `calc.h`.

Other files can call them.

---

### Internal name

An internal name is private to one file.

Use `static` to make a name internal.

Example:

```c
static int sp = 0;
static double val[MAXVAL];
```

These can only be used inside `stack.c`.

---

# stack.c idea

In the book, `stack.c` manages the calculator stack.

It has private variables:

```c
static int sp = 0;
static double val[MAXVAL];
```

Meaning:

```text
sp  = stack position
val = array that stores calculator numbers
```

These should be private.

Other files should not directly change `sp` or `val`.

Other files should use functions:

```c
push(5.0);
pop();
```

So:

```text
sp and val = private data

push and pop = public functions
```

---

## stack.c picture

```text
stack.c
│
├── private static variables
│   ├── sp
│   └── val
│
└── public functions
    ├── push
    └── pop
```

Other files can call `push` and `pop`.

Other files cannot directly touch `sp` and `val`.

---

# getch.c idea

In the book, `getch.c` manages a small input buffer.

It has private variables:

```c
static char buf[BUFSIZE];
static int bufp = 0;
```

Meaning:

```text
buf  = stores characters pushed back
bufp = current buffer position
```

These should be private.

Other files should not directly change `buf` or `bufp`.

Other files should use functions:

```c
getch();
ungetch(c);
```

So:

```text
buf and bufp = private data

getch and ungetch = public functions
```

---

## getch.c picture

```text
getch.c
│
├── private static variables
│   ├── buf
│   └── bufp
│
└── public functions
    ├── getch
    └── ungetch
```

Other files can call `getch` and `ungetch`.

Other files cannot directly touch `buf` and `bufp`.

---

# getop.c idea

`getop.c` reads the next calculator input.

It decides if the input is:

```text
number
operator
newline
EOF
```

`getop.c` uses:

```c
getch();
ungetch(c);
```

But it does not directly touch:

```text
buf
bufp
```

because those are private inside `getch.c`.

---

# Why static is useful here

Without `static`, too many files could directly touch important variables.

That is dangerous.

Example:

```text
main.c should not directly change sp
getop.c should not directly change bufp
```

Instead:

```text
main.c uses push and pop

getop.c uses getch and ungetch
```

This keeps each file responsible for its own private data.

---

# Second use: static inside a function

There is another use of `static`.

```text
static inside a function = remembers value between calls
```

Example:

```c
void countCalls(void) {
    static int count = 0;

    count++;

    printf("%d\n", count);
}
```

Output:

```text
1
2
3
```

A normal local variable would reset every time.

A static local variable does not reset.

---

# Two meanings to remember

```text
static inside a function:
    remembers its value between function calls

static outside a function:
    private to this .c file
```

---

# Main K&R 4.6 lesson

For the book’s calculator program, the important idea is:

```text
static outside a function makes private file data
```

K&R uses it so:

```text
stack.c owns sp and val

getch.c owns buf and bufp

other files use functions instead of touching private variables directly
```

---

# What I need to remember

```text
static = limits access or keeps memory

external name = can be used by other files

internal name = private to one file

static outside a function = internal/private to that file

static inside a function = remembers value between calls

sp and val are private stack data in stack.c

buf and bufp are private buffer data in getch.c

push and pop are public stack functions

getch and ungetch are public input-buffer functions
```
