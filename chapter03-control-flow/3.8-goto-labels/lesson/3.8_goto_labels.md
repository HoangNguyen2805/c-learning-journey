# 3.8 - Goto and Labels

## Big idea

`goto` jumps to another labeled place in the program.

Example:

```c id="a7k82z"
goto done;

done:
    printf("Finished\n");
```

The label is:

```c id="fv0t4t"
done:
```

The jump is:

```c id="7rhqne"
goto done;
```

---

## Label

A label is a name followed by a colon.

```c id="zaa52g"
done:
```

A label marks a place in the code.

---

## Goto

`goto` jumps to a label.

```c id="7uh5k7"
goto done;
```

Meaning:

```text id="opytmx"
jump to the label named done
```

---

## Why goto is dangerous

`goto` can make code hard to read because the program jumps around.

Bad style:

```text id="fsxg19"
jump here
jump there
skip code
hard to follow
```

Beginner rule:

```text id="p2c40h"
Avoid goto unless there is a strong reason.
```

---

## When goto can be useful

K&R says `goto` can sometimes be useful when breaking out of deeply nested loops.

Example idea:

```c id="hmdad5"
for (...) {
    for (...) {
        if (problem)
            goto done;
    }
}

done:
```

This jumps out of both loops at once.

A normal `break` only leaves one loop.

---

## Break vs goto

```text id="yj50s4"
break = leave one loop or switch

goto = jump to a label
```

If you only need to leave one loop, use `break`.

If you need to escape several nested loops, `goto` may be used carefully.
