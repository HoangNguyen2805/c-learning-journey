# 2.12 - Precedence and Order of Evaluation

## Big idea

Precedence means which operator happens first.

Example:

```c
10 + 3 * 2
```

Result:

```text
16
```

because `*` happens before `+`.

---

## Parentheses

Parentheses force the order.

```c
(10 + 3) * 2
```

Result:

```text
26
```

Simple rule:

```text
Use parentheses when the expression is not obvious.
```

---

## Associativity

Associativity means the direction C groups operators when they have the same precedence.

Example:

```c
20 - 5 - 2
```

This groups left to right:

```c
(20 - 5) - 2
```

Result:

```text
13
```

But assignment groups right to left:

```c
a = b = c = 0;
```

Meaning:

```text
c gets 0
b gets 0
a gets 0
```

---

## Common precedence order

Highest to lower:

```text
()                 parentheses
++ --              increment/decrement
* / %              multiplication, division, remainder
+ -                addition, subtraction
< <= > >=          relational
== !=              equality
&                  bitwise AND
^                  bitwise XOR
|                  bitwise OR
&&                 logical AND
||                 logical OR
?:                 conditional expression
= += -= *= /= %=   assignment
```

You do not need to memorize the whole table perfectly yet.

Use parentheses when unsure.

---

## Examples

```c
10 + 3 * 2
```

means:

```c
10 + (3 * 2)
```

```c
10 > 3 && 5 > 2
```

means both comparisons happen before `&&`.

```c
x = 2 + 3 * 4;
```

means:

```c
x = 2 + 12;
```

So:

```text
x = 14
```

---

## Order of evaluation

Order of evaluation is different from precedence.

Precedence tells how expressions are grouped.

Order of evaluation tells which part C calculates first.

K&R warns that C does not always guarantee evaluation order.

---

## Dangerous side effects

A side effect means an expression changes something.

Examples:

```c
i++
i--
x = 5
```

These change a variable.

Avoid changing the same variable more than once in one expression.

Bad:

```c
x = i++ + i++;
```

Bad:

```c
printf("%d %d\n", i, i++);
```

Why bad?

```text
C may not evaluate the parts in the order you expect.
```

Better:

```c
x = i;
i++;
x = x + i;
i++;
```

Simple rule:

```text
Do one change per statement.
```

---

## Function argument order

K&R also warns that function argument evaluation order is not guaranteed.

Avoid code like:

```c
printf("%d %d\n", i++, i++);
```

Write it in separate statements instead.

---

## What I need to remember

```text
precedence = which operator has priority

associativity = grouping direction when operators have same precedence

parentheses make order clear

* / % happen before + -

relational operators happen before logical operators

&& happens before ||

assignment operators group right to left

order of evaluation is not always guaranteed

avoid changing the same variable more than once in one expression

write clear code instead of clever code
```
