# 2.4 - Declarations

## Big idea

A declaration tells C:

```text
what type a variable is
what the variable name is
optional starting value
```

Example:

```c
int age;
```

Meaning:

```text
Create a variable named age.
It stores int values.
```

---

## Declaration with initialization

A variable can get a starting value when it is declared.

```c
int age = 25;
double price = 9.99;
char grade = 'A';
```

This is called initialization.

---

## Declaration vs assignment

Declaration creates the variable:

```c
int age;
```

Assignment gives or changes the value after the variable already exists:

```c
age = 25;
```

Declaration plus initialization does both:

```c
int age = 25;
```

---

## Multiple declarations

You can declare multiple variables of the same type in one line.

```c
int lower, upper, step;
```

This means:

```c
int lower;
int upper;
int step;
```

---

## Declare before use

A variable must be declared before it is used.

Wrong:

```c
age = 25;
int age;
```

Correct:

```c
int age;
age = 25;
```

---

## Uninitialized variables

A local variable without a starting value has an unknown value.

Bad:

```c
int age;
printf("%d\n", age);
```

Better:

```c
int age = 0;
```

Simple rule:

```text
local variable without value = dangerous garbage value
```

---

## External and static variables

K&R says external and static variables are initialized to zero by default if no value is given.

Simple rule:

```text
local variable without value = garbage value
external/static variable without value = starts as 0
```

---

## `const` qualifier

`const` means the value should not be changed after it is initialized.

```c
const int MAX_SCORE = 100;
```

Meaning:

```text
const = qualifier
int = data type
MAX_SCORE = variable name
100 = starting value
```

`const` is not a data type.

It belongs in declarations because it changes how a variable is declared.

---

## What I need to remember

```text
declaration = create variable
initialization = give first value
assignment = give/change value after declaration

variables must be declared before use

local uninitialized variables have garbage values

external/static variables start as 0 by default

const = value should not change after initialization

const is a qualifier, not a data type
```
