# 2.6 - Relational and Logical Operators

## Big idea

Relational and logical operators are used to test conditions.

In C:

```text id="zt6cbv"
0 = false
1 = true
```

Actually, any nonzero value is true.

---

## Relational operators

Relational operators compare two values.

```text id="r7nza6"
>     greater than
>=    greater than or equal to
<     less than
<=    less than or equal to
==    equal to
!=    not equal to
```

Examples:

```c id="wq1f4e"
10 > 3
10 >= 10
3 < 10
3 <= 3
10 == 10
10 != 3
```

Each comparison gives either:

```text id="mtmblt"
1 for true
0 for false
```

---

## `=` vs `==`

These are not the same.

```c id="4h87ls"
x = 10;
```

means:

```text id="eck51v"
assign 10 to x
```

```c id="n56nxh"
x == 10
```

means:

```text id="37m8rq"
check if x equals 10
```

Remember:

```text id="fg81dl"
=  assignment
== comparison
```

---

## Logical operators

Logical operators combine or change conditions.

```text id="3e716g"
&&   AND
||   OR
!    NOT
```

---

## AND: `&&`

Both sides must be true.

```c id="n2mknw"
age >= 18 && age <= 25
```

Meaning:

```text id="6vx16z"
age is at least 18 AND age is at most 25
```

Truth rule:

```text id="eyp5js"
true && true = true
true && false = false
false && true = false
false && false = false
```

---

## OR: `||`

At least one side must be true.

```c id="rgrvty"
grade == 'A' || grade == 'B'
```

Meaning:

```text id="gh0zwe"
grade is A OR grade is B
```

Truth rule:

```text id="0wddwu"
true || true = true
true || false = true
false || true = true
false || false = false
```

---

## NOT: `!`

`!` flips true and false.

```c id="0yh54y"
!done
```

Meaning:

```text id="o58ypn"
not done
```

Examples:

```text id="py5btm"
!0 = 1
!1 = 0
```

---

## Short-circuit evaluation

C evaluates logical expressions from left to right.

For `&&`, if the left side is false, C does not need to check the right side.

```c id="5z5yxa"
0 && something
```

The whole result is false.

For `||`, if the left side is true, C does not need to check the right side.

```c id="fz4zmh"
1 || something
```

The whole result is true.

This is called short-circuit evaluation.

---

## Precedence note

Relational and logical operators have their own order.

Simple beginner rule:

```text id="cy17rc"
arithmetic happens before comparison
comparison happens before && and ||
&& happens before ||
```

Example:

```c id="b89mem"
10 + 5 > 12
```

C does:

```text id="0sa9e7"
10 + 5 first
15 > 12 second
```

Result:

```text id="wvqqia"
1
```

Full precedence is covered later in 2.12.
