# 2.3 - Constants

## Big idea

A constant is a fixed value written directly in the program.

Examples:

```c
100
3.14
'A'
"Hoang"
'\n'
```

K&R 2.3 is mostly about constant values, not `const` variables.

---

## Integer constants

Integer constants are whole-number values.

```c
31
100
0
-10
```

C can write integer constants in different bases:

```c
31      // decimal
037     // octal
0x1F    // hexadecimal
```

Meaning:

```text
decimal = normal base 10
octal = base 8, starts with 0
hexadecimal = base 16, starts with 0x or 0X
```

---

## Integer suffixes

Integer constants can use suffixes:

```c
123L     // long
100U     // unsigned
100UL    // unsigned long
```

Common suffixes:

```text
L or l = long
U or u = unsigned
UL or ul = unsigned long
```

Use uppercase `L` because lowercase `l` looks like `1`.

---

## Floating-point constants

Floating-point constants are decimal values.

```c
3.14
0.5
10.0
```

They can also use scientific notation:

```c
1.5e3
```

Meaning:

```text
1.5e3 = 1.5 × 10^3 = 1500
```

Floating constants can also use suffixes:

```c
3.14F    // float
3.14L    // long double
```

---

## Character constants

A character constant uses single quotes.

```c
'A'
'7'
'$'
```

Example:

```c
char grade = 'A';
```

A character constant represents one character value.

---

## Escape sequences

Some characters are hard to type directly, so C uses escape sequences.

An escape sequence starts with backslash `\`.

Common escape sequences:

```text
\n    newline
\t    horizontal tab
\v    vertical tab
\b    backspace
\r    carriage return
\f    form feed
\a    alert / bell
\\    backslash
\?    question mark
\'    single quote
\"    double quote
\ooo  octal character value
\xhh  hexadecimal character value
```

Examples:

```c
char newline = '\n';
char tab = '\t';
char quote = '\"';
char backslash = '\\';
```

K&R also shows symbolic names for escape characters:

```c
#define VTAB '\013'
#define BELL '\007'
```

Meaning:

```text
VTAB = vertical tab
BELL = alert / bell
```

Modern C can write those more clearly:

```c
#define VTAB '\v'
#define BELL '\a'
```

---

## String constants

A string constant uses double quotes.

```c
"Hello"
"Hoang"
"Chapter 2"
```

A string is stored as a character array ending with `'\0'`.

Example:

```c
"Hi"
```

is stored like:

```text
'H' 'i' '\0'
```

Important:

```text
'A' = character constant
"A" = string constant
```

---

## Constant expressions

A constant expression uses only constants.

Example:

```c
#define MAXLINE 1000
char line[MAXLINE + 1];
```

`MAXLINE + 1` can be evaluated before the program runs because it uses constant values.

---

## Enumeration constants

An `enum` creates named integer constants.

Example:

```c
enum boolean {
    NO,
    YES
};
```

By default:

```text
NO = 0
YES = 1
```

You can also give values manually:

```c
enum months {
    JAN = 1,
    FEB,
    MAR
};
```

Then:

```text
JAN = 1
FEB = 2
MAR = 3
```

---

## Symbolic constants with `#define`

A symbolic constant gives a name to a constant value.

Example:

```c
#define MAX_SCORE 100
#define PASSING_SCORE 70
```

Rules:

```text
No =
No semicolon
Usually uppercase name
```

Correct:

```c
#define MAX_SCORE 100
```

Wrong:

```c
#define MAX_SCORE = 100;
```

---

## Note about `const`

`const` is not a data type.

It is a qualifier used in declarations.

Example:

```c
const int MAX_SCORE = 100;
```

Meaning:

```text
const = qualifier
int = data type
MAX_SCORE = variable name
100 = constant value
```

K&R 2.3 focuses on constant values like `100`, `'A'`, `"text"`, `enum`, and `#define`.

We will study `const` more in 2.4 Declarations.

---

## What I need to remember

```text
constant = fixed value written in code

31 = decimal integer constant
037 = octal integer constant
0x1F = hexadecimal integer constant

L = long
U = unsigned
UL = unsigned long

3.14 = floating-point constant
1.5e3 = scientific notation

'A' = character constant
"Hoang" = string constant

escape sequences start with \

\n = newline
\t = tab
\a = alert / bell
\v = vertical tab

#define NAME value = symbolic constant

enum = named integer constants

const is a qualifier, not a data type
```
