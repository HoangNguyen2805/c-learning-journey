# Chapter 5.4 — Address Arithmetic

## Overview

This chapter dives deep into pointer operations: arithmetic on pointers, dynamic memory allocation (malloc/free), the static keyword, and storage classes. These are the tools to manage memory manually in C.

## Part 1: Pointer Arithmetic (Review & Deep Dive)

### How Pointer Arithmetic Scales

When you do `p + 1`, the compiler doesn't add 1 byte — it adds `sizeof(*p)` bytes.

```c
int *ip = arr;      // points to arr[0]
ip + 1              // points to arr[1], NOT arr[0] + 1 byte
                    // actually arr[0] + 4 bytes (on most systems)

char *cp = str;     // points to str[0]
cp + 1              // points to str[1], NOT str[0] + 1 byte
                    // actually str[0] + 1 byte
```

**Why?** So you navigate arrays element-by-element, not byte-by-byte.

### Pointer Subtraction

You can subtract pointers of the same type:

```c
int arr[5] = {10, 20, 30, 40, 50};
int *p1 = &arr[1];  // points to arr[1] (20)
int *p2 = &arr[4];  // points to arr[4] (50)

int distance = p2 - p1;  // result: 3 (elements between them)
```

**Result is the number of elements between the pointers**, not bytes.

### Pointer Comparisons

You can compare pointers:

```c
int arr[5] = {1, 2, 3, 4, 5};
int *p1 = &arr[0];
int *p2 = &arr[3];

if (p1 < p2) {      // TRUE — p1 points earlier in array
    printf("p1 comes before p2\n");
}

if (p1 == p2) {     // FALSE — different addresses
    printf("Same address\n");
}
```

Valid comparisons: `<`, `<=`, `>`, `>=`, `==`, `!=`

## Part 2: Dynamic Memory Allocation

### What is Dynamic Memory?

**Static memory** (stack): fixed size at compile time
```c
int arr[100];  // size decided at compile time, fixed
```

**Dynamic memory** (heap): size decided at runtime
```c
int *arr = malloc(n * sizeof(int));  // size decided at runtime, flexible
```

### malloc() — Allocate Memory

```c
int *p = malloc(sizeof(int));  // allocate space for 1 int
```

**What malloc does:**
1. Allocates memory on the heap
2. Returns a **pointer** to that memory
3. Returns NULL if allocation fails

**Example:**
```c
int *p = malloc(sizeof(int));
if (p == NULL) {
    printf("Memory allocation failed\n");
    return 1;
}
*p = 42;  // use the allocated memory
```

### Allocating Arrays Dynamically

```c
int size = 10;
int *arr = malloc(size * sizeof(int));  // allocate space for 10 ints

for (int i = 0; i < size; i++) {
    arr[i] = i * 10;  // use like normal array
}
```

### free() — Deallocate Memory

When done with dynamically allocated memory, **free it**:

```c
int *p = malloc(sizeof(int));
*p = 42;
free(p);  // release memory back to system
p = NULL; // good practice: set to NULL after freeing
```

**Why free?** Memory is a finite resource. Not freeing causes **memory leak**.

**Why set to NULL?** Prevents accidental use of freed memory (dangling pointer).

### Memory Leak Example

```c
void bad_function() {
    int *p = malloc(sizeof(int));
    *p = 42;
    // forgot to free(p)
    return;  // p is leaked — memory never returned
}

void good_function() {
    int *p = malloc(sizeof(int));
    *p = 42;
    free(p);  // properly released
    p = NULL;
    return;
}
```

## Part 3: The static Keyword

### static with Local Variables

```c
void counter() {
    static int count = 0;  // initialized once, persists
    count++;
    printf("Count: %d\n", count);
}

int main() {
    counter();  // prints: Count: 1
    counter();  // prints: Count: 2
    counter();  // prints: Count: 3
    return 0;
}
```

**Key:** `static` local variables are initialized **once** and **persist** across function calls.

### static with Global Variables

```c
static int global_var = 10;  // file-scoped (not visible to other files)

void modify() {
    global_var++;
}
```

**Key:** `static` globals are visible only within the same file (internal linkage).

### static with Functions

```c
static void helper() {  // visible only in this file
    printf("Helper\n");
}

void public_func() {    // visible to other files (external linkage)
    helper();
}
```

## Part 4: Storage Classes

C has four storage classes: **auto**, **register**, **static**, **extern**

### auto (default for locals)

```c
{
    auto int x = 5;  // same as: int x = 5;
}
```

**Scope:** block (disappears at end of block)
**Lifetime:** stack

### register

```c
register int i;  // suggest to compiler: keep in CPU register
for (int i = 0; i < 1000000; i++) {
    // i is accessed many times
}
```

**Scope:** block
**Lifetime:** stack
**Note:** just a suggestion to compiler; modern compilers ignore it.

### static

Already covered above.

### extern

```c
// file1.c
int global_count = 0;

// file2.c
extern int global_count;  // declare external variable
void increment() {
    global_count++;  // use it
}
```

**Scope:** entire program (across files)
**Lifetime:** program lifetime

## Part 5: Combining Pointers and Allocation

### Pointer to Dynamic Array

```c
int size;
printf("Array size: ");
scanf("%d", &size);

int *arr = malloc(size * sizeof(int));
if (arr == NULL) {
    printf("Failed to allocate\n");
    return 1;
}

for (int i = 0; i < size; i++) {
    arr[i] = i * 2;
}

// Use as normal array
for (int i = 0; i < size; i++) {
    printf("%d ", arr[i]);
}

free(arr);  // MUST free when done
arr = NULL;
```

### Pointer to Pointer (for functions that allocate)

```c
void allocate_array(int **arr, int size) {
    *arr = malloc(size * sizeof(int));
}

int main() {
    int *my_arr;
    allocate_array(&my_arr, 10);  // pass address of pointer
    
    my_arr[0] = 42;
    free(my_arr);
    my_arr = NULL;
    return 0;
}
```

**Why double pointer?** Function needs to modify the pointer itself, not just what it points to.

## Part 6: Common Mistakes

### 1. Forgetting to Check malloc Return

```c
// BAD
int *p = malloc(sizeof(int));
*p = 42;  // crashes if malloc returned NULL

// GOOD
int *p = malloc(sizeof(int));
if (p == NULL) {
    printf("Allocation failed\n");
    return 1;
}
*p = 42;
```

### 2. Memory Leak (Forgetting free)

```c
// BAD
void leak() {
    int *p = malloc(sizeof(int));
    *p = 42;
    return;  // memory leaked
}

// GOOD
void no_leak() {
    int *p = malloc(sizeof(int));
    *p = 42;
    free(p);  // released
    p = NULL;
}
```

### 3. Use After Free (Dangling Pointer)

```c
// BAD
int *p = malloc(sizeof(int));
*p = 42;
free(p);
printf("%d\n", *p);  // USE AFTER FREE — undefined behavior

// GOOD
int *p = malloc(sizeof(int));
*p = 42;
free(p);
p = NULL;
// don't use p anymore
```

### 4. Freeing Wrong Size

```c
// BAD
int *p = malloc(10 * sizeof(int));
// ... use it ...
free(p);  // fine (size doesn't matter to free)

// BETTER practice: remember allocation size
int size = 10;
int *p = malloc(size * sizeof(int));
// ...
free(p);
```

## Summary

| Concept | Key Point |
|---------|-----------|
| **Pointer arithmetic** | `p + 1` moves by `sizeof(*p)` bytes, not 1 byte |
| **malloc** | Allocates memory, returns pointer, returns NULL on failure |
| **free** | Deallocates memory, prevents leaks |
| **static** | Local: persists across calls; global: file-scoped |
| **Storage classes** | auto, register, static, extern control scope and lifetime |
| **Dangling pointer** | Don't use after free; set to NULL |
