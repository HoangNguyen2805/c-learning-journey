# Chapter 5.3 — Pointers and Arrays

## Overview

Arrays and pointers are fundamentally related in C. An array name is essentially a constant pointer to its first element. This chapter explores:
- How arrays decay to pointers
- Pointer arithmetic with arrays
- Equivalence of `arr[i]` and `*(arr+i)`
- Multi-dimensional arrays and pointers
- String arrays

## Key Concepts

### Arrays Decay to Pointers

When you declare an array:
```c
int arr[5] = {1, 2, 3, 4, 5};
```

The name `arr` is a **constant pointer** to `arr[0]`. It decays to a pointer in most contexts.

- `arr` ≡ `&arr[0]` (address of first element)
- You cannot reassign `arr` to point elsewhere (it's constant)
- `arr[i]` works because it's internally converted to pointer arithmetic

### Pointer Arithmetic

When you perform arithmetic on a pointer, the compiler scales by the size of what it points to:

```c
int *p = arr;  // p points to arr[0]
p + 1          // points to arr[1] (moves 4 bytes forward on most systems)
p + 2          // points to arr[2] (moves 8 bytes forward)
```

For `char *` pointers:
```c
char *cp = "hello";
cp + 1         // moves 1 byte forward (chars are 1 byte)
```

### Equivalence: arr[i] vs *(arr+i)

These two are **exactly equivalent**:

```c
arr[i]      // array subscript notation
*(arr + i)  // pointer arithmetic and dereference
```

Both:
1. Take the address of arr[0]
2. Add i * sizeof(int) bytes
3. Dereference to get the value

The compiler converts `arr[i]` to `*(arr+i)` internally.
