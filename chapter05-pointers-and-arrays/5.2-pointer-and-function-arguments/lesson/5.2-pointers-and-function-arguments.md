# 5.2 - Pointers and Function Arguments

## Big idea

In Chapter 5.1, you learned that **pointers let functions modify their caller's variables**.

Chapter 5.2 is about putting that idea to work: **passing pointers to functions so they can reach into your variables and change them**.

This is the foundation for:
- Functions that return multiple values (by modifying arguments)
- Arrays (which are closely related to pointers)
- String manipulation
- Dynamic data structures (later chapters)

---

## The problem pointers solve

**Without pointers**, functions receive **copies** of arguments:

```c
void increment(int x) {
    x = x + 1;  // this only changes the local copy
}

int main() {
    int n = 5;
    increment(n);
    printf("%d\n", n);  // still 5, NOT 6
}
```

The function increments a **copy** of n, not n itself.

**With pointers**, the function can modify the **original**:

```c
void increment(int *x) {
    *x = *x + 1;  // this changes the original variable
}

int main() {
    int n = 5;
    increment(&n);  // pass the address
    printf("%d\n", n);  // now 6
}
```

**Why this matters**: Many real-world functions need to "return" multiple values or modify their arguments. Without pointers, you can't do this.

## Function arguments and pointers

### Pattern 1: Function modifies a single variable

```c
void addFive(int *ptr) {
    *ptr = *ptr + 5;  // modify what ptr points to
}

int main() {
    int x = 10;
    addFive(&x);      // pass x's address
    printf("%d\n", x);  // x is now 15
    return 0;
}
```

**Key:** Pass the **address** (`&x`), not the value.

### Pattern 2: Function returns status, modifies arguments

Functions often return a success/failure code and use arguments to "return" results:

```c
// Returns 1 if successful, 0 if error
// Stores the result in the pointer argument
int divide(int a, int b, int *result) {
    if (b == 0) {
        return 0;  // error: division by zero
    }
    *result = a / b;
    return 1;  // success
}

int main() {
    int ans;
    if (divide(10, 2, &ans)) {
        printf("Result: %d\n", ans);  // 5
    } else {
        printf("Error: division by zero\n");
    }
    return 0;
}
```

This pattern is everywhere in C — especially for error handling.

## Arrays and pointers

This is the **deepest connection** between arrays and pointers:

**An array name is essentially a pointer to its first element.**

```c
int arr[5] = {10, 20, 30, 40, 50};
int *p = arr;  // p points to arr[0]

printf("%d\n", *p);     // 10 (first element)
printf("%d\n", *(p+1)); // 20 (second element)
printf("%d\n", p[0]);   // 10 (same as *p)
printf("%d\n", p[1]);   // 20 (same as *(p+1))
```

**Key insight**: `arr[i]` is just shorthand for `*(arr + i)`.

When you write `arr`, the compiler treats it as a pointer to the first element.

### Passing arrays to functions

When you pass an array to a function, you're really passing a **pointer**:

```c
void printArray(int *arr, int size) {
    for (int i = 0; i < size; i++) {
        printf("%d ", arr[i]);  // or *(arr + i)
    }
    printf("\n");
}

int main() {
    int numbers[5] = {1, 2, 3, 4, 5};
    printArray(numbers, 5);  // arr decays to a pointer
    return 0;
}
```

**Important**: When you pass an array, it **decays to a pointer to the first element**. Inside the function, you get a pointer, not a copy of the array.

This is why:
- Modifying an array inside a function **modifies the original** (no copy is made)
- You need to pass the size separately (the function doesn't know the array's length)

### Modifying arrays through pointers

Since the function receives a pointer to the original array, it can modify it:

```c
void doubleValues(int *arr, int size) {
    for (int i = 0; i < size; i++) {
        arr[i] = arr[i] * 2;  // modifies the original array
    }
}

int main() {
    int numbers[3] = {1, 2, 3};
    doubleValues(numbers, 3);
    printf("%d %d %d\n", numbers[0], numbers[1], numbers[2]);  // 2 4 6
    return 0;
}
```

## Strings and pointers

In C, **strings are arrays of characters**.

```c
char str[20] = "hello";
```

This creates an array of 5 characters + a null terminator `\0`.

A function that works with strings receives a **pointer to the first character**:

```c
int strlen(char *s) {
    int count = 0;
    while (*s != '\0') {  // loop until null terminator
        count++;
        s++;  // move to next character
    }
    return count;
}

int main() {
    char name[50] = "Claude";
    printf("%d\n", strlen(name));  // 6
    return 0;
}
```

**Key**: Strings are terminated by a null character `\0`, so functions know where the string ends.

### Pointer arithmetic

Notice `s++` in the strlen example — **you can do arithmetic on pointers**:

```c
char *s = "hello";
printf("%c\n", *s);      // 'h'
printf("%c\n", *(s+1));  // 'e'
printf("%c\n", *(s+2));  // 'l'

s++;  // move s to point to the next character
printf("%c\n", *s);      // 'e'
```

`s++` moves the pointer forward by one element. For `char *`, it moves by 1 byte. For `int *`, it moves by 4 bytes (the size of an int).

This is a powerful pattern for iterating through arrays and strings.

## Common patterns

### Pattern: "Return" multiple values

```c
void getMinMax(int *arr, int size, int *min, int *max) {
    *min = *max = arr[0];
    for (int i = 1; i < size; i++) {
        if (arr[i] < *min) *min = arr[i];
        if (arr[i] > *max) *max = arr[i];
    }
}

int main() {
    int numbers[] = {3, 7, 2, 9, 1};
    int min, max;
    getMinMax(numbers, 5, &min, &max);
    printf("Min: %d, Max: %d\n", min, max);  // Min: 1, Max: 9
    return 0;
}
```

Function modifies two variables through pointers.

### Pattern: Modify an array in-place

```c
void reverse(int *arr, int size) {
    for (int i = 0; i < size / 2; i++) {
        int temp = arr[i];
        arr[i] = arr[size - 1 - i];
        arr[size - 1 - i] = temp;
    }
}

int main() {
    int numbers[] = {1, 2, 3, 4, 5};
    reverse(numbers, 5);
    for (int i = 0; i < 5; i++) printf("%d ", numbers[i]);  // 5 4 3 2 1
    return 0;
}
```

### Pattern: Copy or compare strings

```c
int streq(char *s1, char *s2) {
    while (*s1 == *s2) {
        if (*s1 == '\0') return 1;  // both ended at same point
        s1++;
        s2++;
    }
    return 0;
}

int main() {
    if (streq("hello", "hello")) {
        printf("Strings are equal\n");
    }
    return 0;
}
```

Using pointer arithmetic to walk through strings.

## Key points to remember

```text
When you pass a pointer to a function, the function can modify
the original variable (the one the pointer points to).

Arrays decay to pointers to their first element when passed to functions.

Inside a function receiving an array, you get a pointer, not a copy.
Modifying the array modifies the original.

Strings are null-terminated arrays of characters.

Pointer arithmetic (p++, p+i) moves the pointer by the size of
what it points to (1 byte for char*, 4 bytes for int*, etc.).

These patterns let you:
- Have functions modify multiple variables
- Work with arrays and strings efficiently
- Build complex data structures
```

## Common gotchas

### Gotcha 1: Forgetting to pass the address

```c
void increment(int *x) {
    *x = *x + 1;
}

int n = 5;
increment(n);      // WRONG: passes value, not address
increment(&n);     // RIGHT: passes address
```

### Gotcha 2: Confusing array names with pointers

Array names **decay** to pointers in most contexts, but they're not quite the same:

```c
int arr[5] = {1, 2, 3, 4, 5};
int *p = arr;  // arr decays to pointer to arr[0]

// These are equivalent:
printf("%d\n", arr[0]);
printf("%d\n", *arr);
printf("%d\n", *p);

// But arr can't be reassigned:
// arr = p;  // ERROR
// p = arr;  // OK
```

### Gotcha 3: Not passing array size

When you pass an array to a function, you only pass a pointer to the first element. The function has **no idea how many elements** are in the array:

```c
void printArray(int *arr) {
    // How big is arr? The function doesn't know!
    // This is why you need to pass the size separately
}

printArray(numbers, size);  // Always pass size too
```

## Summary

5.2 is about **using pointers as function arguments** to:
- Let functions modify caller's variables
- Work with arrays and strings
- "Return" multiple values

The core pattern: **pass the address (`&variable` or `array`), use the pointer (`*ptr` or `ptr[i]`) inside the function.**

This unlocks the real power of C — functions that can modify data outside their scope.
