# 5.1 - Pointers and Addresses

## Big idea

A **pointer** is a variable whose value is a **memory address**.

That's it — that's the whole concept. A normal variable stores a value (like `5` or `'a'`). A pointer variable stores the **location in memory** where a value lives.

Think of your computer's memory as a giant street of numbered houses. Every variable you create lives in one of those houses (has an address). A pointer is like a piece of paper where you've written down someone's house address — the paper itself isn't the house, it just tells you where the house is.

---

## Why pointers matter

Before pointers, everything you've done in C has worked with **copies** of values:

```c
void tryToChange(int x) {
    x = 100;  // this only changes the LOCAL copy
}

int main() {
    int n = 5;
    tryToChange(n);
    printf("%d\n", n);  // still prints 5, NOT 100
}
```

`tryToChange` receives a **copy** of `n`. Changing the copy does nothing to the original.

Pointers solve this. Instead of passing a copy of the value, you pass the **address** of the variable, so the function can go find and change the *original*.

This is the foundation for:
- Functions that modify their caller's variables (like `scanf(&x)` — you've been using this all along!)
- Arrays (an array name is closely related to a pointer to its first element)
- Dynamic memory, linked lists, trees — most "real" data structures later in your C journey

---

## The two core operators

There are exactly two operators you need to understand pointers:

### `&` — "address of"

Given a variable, `&` gives you **its address in memory**.

```c
int x = 5;
printf("%p\n", &x);  // prints something like 0x7ffee3ab5c9c
```

`&x` doesn't mean "the value of x" — it means "**where** x lives in memory."

### `*` — "dereference" / "go to this address"

Given an address, `*` lets you **go to that address and access what's stored there**.

```c
int x = 5;
int *p = &x;      // p now holds the ADDRESS of x
printf("%d\n", *p); // *p means "go to the address p holds, and get the value there" -> prints 5
```

**Key insight:** `&` and `*` are opposites of each other:
- `&x` — "give me the address of this value"
- `*p` — "give me the value at this address"

---

## Declaring a pointer

```c
int *p;
```

Read this as: "`p` is a variable that holds the address of an `int`."

The `*` in a **declaration** doesn't mean "dereference" — it's just part of the syntax that says "this variable is a pointer." Once declared, using `*p` elsewhere (outside a declaration) means "dereference."

```c
int *p;      // p is a pointer to an int
char *c;     // c is a pointer to a char
double *d;   // d is a pointer to a double
```

**The type matters.** `int *p` means `p` can only validly point to `int` variables. This tells the compiler how many bytes to read/write when you dereference `p`, and how to interpret those bytes.

---

## Putting it together: a full example

```c
#include <stdio.h>

int main() {
    int x = 5;
    int *p;      // declare p as a pointer to an int
    
    p = &x;      // p now holds the address of x
    
    printf("x = %d\n", x);         // 5
    printf("&x = %p\n", &x);       // address of x, e.g. 0x7ffee3ab5c9c
    printf("p = %p\n", p);         // same address as &x
    printf("*p = %d\n", *p);       // 5 (dereferencing p gets x's value)
    
    *p = 100;    // changes the value AT the address p points to
    
    printf("x = %d\n", x);         // 100! Changing *p changed x itself
    
    return 0;
}
```

**Output:**
```
x = 5
&x = 0x7ffee3ab5c9c
p = 0x7ffee3ab5c9c
*p = 5
x = 100
```

Notice: `p` and `&x` are the exact same address. `*p` and `x` are the exact same value, at all times — because `*p` literally means "go look at what's at that address," and that address *is* where `x` lives.

When you do `*p = 100;`, you're not changing `p` (the address stays the same) — you're changing **what's stored at that address**, which is `x`.

---

## Visualizing it

```
Memory:
┌──────────────┬─────────┬────────────┐
│  Variable    │  Value  │  Address   │
├──────────────┼─────────┼────────────┤
│  x           │    5    │  0x1000    │
│  p           │  0x1000 │  0x2000    │
└──────────────┴─────────┴────────────┘

p's VALUE is 0x1000 — the address of x.
*p means "go to address 0x1000, and get what's there" → 5
&x means "give me the address of x" → 0x1000
```

`p` itself lives somewhere in memory too (at `0x2000` in this example) — but what `p` **stores** is the number `0x1000`, which happens to be x's address.

---

## Why this matters for function calls

Remember `scanf`?

```c
int n;
scanf("%d", &n);
```

You've been using `&` this whole time! `scanf` needs the **address** of `n` so it can reach into your variable and store the value it read — it can't just receive a copy, because then it couldn't change your actual variable.

This is the exact same principle. Now you know *why* `&n` is required — without it, `scanf` would only get a copy of `n`'s current (garbage) value, with no way to write the new input back into your variable.

## Common mistakes (gotchas)

### 1. Uninitialized pointers

```c
int *p;
*p = 5;   // DANGER: p doesn't point anywhere valid yet!
```

`p` was declared but never assigned an address. It could be pointing at **anything** — some random location in memory. Writing through it can crash your program or corrupt other data. Always assign a pointer before dereferencing it:

```c
int x;
int *p = &x;   // now p points somewhere valid
*p = 5;        // safe
```

### 2. Confusing `*` in declaration vs. dereference

```c
int *p;     // HERE, * means "p is a pointer" (declaration)
*p = 5;     // HERE, * means "dereference p" (go to that address)
```

Same symbol, different meaning depending on context. Declarations vs. usage.

### 3. Mismatched types

```c
int x = 5;
double *p = &x;   // WRONG: p is a pointer to double, but x is an int
```

The compiler will warn you about this — pointer types should match the variable type they're pointing to.

## What to remember

```text
A pointer is a variable that stores a memory address.

&variable   = "address of" this variable
*pointer    = "dereference" — go to the address and get/set the value there

Declaration:  int *p;      -> p is declared as a pointer to an int
Assignment:   p = &x;      -> p now holds x's address
Dereference:  *p           -> the value stored at the address p holds

*p and x are the SAME value at all times, because *p literally means
"go look at x's location."

Always initialize a pointer (point it somewhere valid) before dereferencing it.

This is why scanf needs &n — it needs the address to modify your variable directly.
```
